Send Transactional Email

Event-triggered email delivery with Handlebars templates, idempotent send keys, provider retries, suppression list for bounces and complaints, and a full delivery log. Provider-agnostic adapter.

Overview

Event-triggered email delivery with Handlebars templates, idempotent send keys, provider retries, suppression list for bounces and complaints, and a full delivery log. Provider-agnostic adapter.

Prerequisites

This block has no prerequisites — it can be implemented standalone.

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

email_templates
Handlebars subject/body templates, per-shop overrides or platform defaults, active flag.
email_log
Every send attempt: idempotency_key, status, provider_message_id, error, metadata.
email_suppressions
Bounces, complaints, manual suppression. Recipients are blocked from further sends.

Use it / Avoid it

Use when
  • App needs event-driven email (order placed, signup, shipping).
  • Mentions “email”, “notification”, “order confirmation”.
  • Merchant wants to manage templates via admin UI.
Avoid when
  • Marketing or bulk email campaigns — use a marketing block.
  • SMS or push notifications.
  • In-app notifications only with no email.
