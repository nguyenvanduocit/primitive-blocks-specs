Shopify GDPR Mandatory Webhooks

The three GDPR webhook endpoints every Shopify App Store submission must implement: customers/data_request, customers/redact, and shop/redact. Missing or wrong implementation = rejected review.

Overview

The three GDPR webhook endpoints every Shopify App Store submission must implement: customers/data_request, customers/redact, and shop/redact. Missing or wrong implementation = rejected review.

Prerequisites

This block depends on the following blocks being implemented first:

webhooks.shopify-webhooks

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

gdpr_requests
Audit trail with request_type enum, idempotency by shopify_request_id, status state machine. Survives shop deletion via ON DELETE SET NULL.

Use it / Avoid it

Use when
  • Any app submitted to the Shopify App Store (mandatory).
  • Mentions “GDPR”, “privacy webhooks”, “data erasure”.
  • App stores any customer PII.
Avoid when
  • Building a Shopify theme — not subject to these webhooks.
  • Internal tooling never published to the App Store.
  • Apps that provably store zero customer data (rare in practice).
