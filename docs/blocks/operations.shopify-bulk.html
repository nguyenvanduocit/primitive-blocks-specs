Shopify Bulk Operations

Async bulk query and mutation pipeline. Submit a GraphQL operation to Shopify, listen for the completion webhook, download the JSONL result, and process records with the parent-child convention — without exhausting rate limits.

Overview

Async bulk query and mutation pipeline. Submit a GraphQL operation to Shopify, listen for the completion webhook, download the JSONL result, and process records with the parent-child convention — without exhausting rate limits.

Prerequisites

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

bulk_operations
Lifecycle of every submitted operation: type, status, query_text, result_url, error_code, object_count, file_size.

Use it / Avoid it

Use when
  • Read or write more than ~200 records at once.
  • Mentions “bulk export/import”, “JSONL”, “bulkOperationRunQuery”.
  • Need to sync the full Shopify catalog to an external system.
Avoid when
  • Fewer than ~200 records — use paginated queries.
  • Real-time operations — bulk ops are async (minutes to hours).
  • Per-record error handling — bulk mutation errors aggregate.
