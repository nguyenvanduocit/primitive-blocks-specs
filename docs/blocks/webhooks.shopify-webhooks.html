Shopify Webhook Management

Subscribe to Shopify events, verify each delivery with HMAC-SHA256, and process each payload exactly once using the X-Shopify-Webhook-Id idempotency key. The event backbone other blocks (GDPR) depend on.

Overview

Subscribe to Shopify events, verify each delivery with HMAC-SHA256, and process each payload exactly once using the X-Shopify-Webhook-Id idempotency key. The event backbone other blocks (GDPR) depend on.

Prerequisites

This block depends on the following blocks being implemented first:

auth.shopify-oauth

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

webhook_subscriptions
Per-shop topic subscription with callback URL and Shopify GraphQL GID.
webhook_deliveries
Idempotent delivery log keyed by X-Shopify-Webhook-Id with status state machine and forensic payload hash.

Use it / Avoid it

Use when
  • App needs to react to Shopify events in real time.
  • Mentions “webhook”, “orders create”, “APP_UNINSTALLED”.
  • Downstream blocks like compliance.shopify-gdpr require this.
Avoid when
  • One-time data sync — use operations.shopify-bulk.
  • Storefront data without real-time need — poll the Admin API.
  • Building a Shopify theme.
