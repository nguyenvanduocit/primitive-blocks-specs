Shopify App Proxy

Serve app-generated content (HTML, JSON, or Liquid) under the merchant’s own storefront domain. Shopify forwards proxied requests — the app verifies the signature and responds with the appropriate content type.

Overview

Serve app-generated content (HTML, JSON, or Liquid) under the merchant’s own storefront domain. Shopify forwards proxied requests — the app verifies the signature and responds with the appropriate content type.

Prerequisites

This block depends on the following blocks being implemented first:

auth.shopify-oauth

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

shops
Read-only lookup by shop_domain query param. Owned by auth.shopify-oauth.

Use it / Avoid it

Use when
  • App surfaces content on the storefront, not just the admin.
  • Mentions “storefront widget”, “liquid template”, “/apps/ URL”.
  • Content needs to live under the merchant’s domain.
Avoid when
  • Content only needed in the admin.
  • Building a headless storefront — use Storefront API directly.
  • Need customer auth on the endpoint — proxy is always public.
