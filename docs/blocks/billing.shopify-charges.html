Shopify App Billing & Subscriptions

Recurring subscriptions, one-time charges, and usage-based billing through the Shopify Billing API — the only allowed channel to monetize a Shopify App Store listing. Handles the approve-activate lifecycle and feature gating.

Overview

Recurring subscriptions, one-time charges, and usage-based billing through the Shopify Billing API — the only allowed channel to monetize a Shopify App Store listing. Handles the approve-activate lifecycle and feature gating.

Prerequisites

This block depends on the following blocks being implemented first:

auth.shopify-session-token

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

billing_plans
Plan catalog: price, currency, interval, trial days, feature flags, sort order, active flag.
shop_subscriptions
Per-shop subscription lifecycle with status state machine and Shopify charge GID.
usage_records
Idempotent usage-based charges, keyed by idempotency_key against Shopify usage GID.

Use it / Avoid it

Use when
  • App needs to charge merchants for access.
  • Mentions “billing”, “subscription”, “monetize”, “trial”, “usage-based”.
  • App needs feature gating by subscription tier.
Avoid when
  • Building a free app with no monetization.
  • Charging shoppers — that’s the Shopify Payments API.
  • Non-Shopify payment processor — App Store requires Shopify Billing.
