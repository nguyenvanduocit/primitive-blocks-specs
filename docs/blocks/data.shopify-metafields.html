Shopify Metafields

Overview

Attach app-defined custom data to Shopify resources (products, orders, customers, shop). Definitions live locally for validation; values live in Shopify via GraphQL — the source of truth never diverges.

Prerequisites

This block depends on the following blocks being implemented first:

auth.shopify-session-token

Data Model

Logical tables introduced by this block. Types resolve to dialect-specific SQL per the canonical logical-type mapping.

metafield_definitions
Local registry of definitions registered in Shopify (namespace, key, owner_type, type). Values themselves stay in Shopify.

Use it / Avoid it

Use when
  • App needs to extend a Shopify resource with custom attributes.
  • Mentions “metafields”, “custom data”, “extend Shopify model”.
  • Data logically belongs to a Shopify resource, not your own domain.
Avoid when
  • App-internal data with no Shopify resource relationship.
  • Large blobs or binary data — use file uploads.
  • Data requiring complex querying — metafield filtering is limited.
